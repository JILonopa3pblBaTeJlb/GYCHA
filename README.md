# 📼 Radio GYCHA: Ultra-Automated Streaming Engine 📡

**Radio GYCHA** — это высокотехнологичный программный стек на базе Python и FFmpeg, предназначенный для создания автономных трансляций. 

---

## 🏛 1. Архитектурная философия системы

Проект построен на модульной архитектуре, где каждый компонент отвечает за отдельный аспект вещания. Система разделена на четыре ключевых слоя:

1.  **Orchestration Layer (Управление):** Жизненный цикл эфира и работа с процессами.
2.  **Graphics Engine (UI):** Рендеринг текстового интерфейса и сбор метрик.
3.  **Content Pipeline (Медиа):** Сбор, валидация и подготовка плейлистов.
4.  **Interactive & Feedback (Связь):** Взаимодействие со зрителями и мониторинг.

---

## ⚙️ 2. Глубокий разбор модулей

### 🧠 Слой оркестрации и стабильности (Core)
*   **`air_1h.py` (The Orchestrator):** Центральный мозг системы. Каждые 60 минут он инициирует сборку нового плейлиста, проверяет расписание на наличие спец-эфиров (VHS) и осуществляет бесшовный перезапуск FFmpeg. Он знает, когда нужно "подхватить" эфир с середины часа, если система была перезагружена.
*   **`process_manager.py` (Process Guard):** Интеллектуальная обертка над подпроцессами. В отличие от простого запуска, этот модуль анализирует `stderr` поток FFmpeg. Если данные перестают поступать (зависание энкодера или сетевого сокета), менеджер принудительно убивает процесс и перезапускает его. Реализует логику "мягкой остановки" через `SIGINT` (имитация нажатия 'q').
*   **`air_supply.py` (Time Engine):** Утилитный модуль для прецизионного расчета таймингов. Отвечает за вычисление "следующего события" (переключение часа или старт фильма) и замер длительности видео через `ffprobe`.
*   **`config_loader.py` (Hot-Reload Config):** Синглтон-объект конфигурации. Позволяет менять настройки `config.ini` прямо во время вещания (например, сменить битрейт или список городов для погоды) без перезапуска основных скриптов.

### 📺 Графический движок и ASCII-интерфейс
*   **`status.py` (The Renderer):** Основной цикл отрисовки (1 FPS). Собирает данные из словаря состояния системы и формирует ASCII-кадр. Реализует систему слоев: левая панель (ресурсы, плейлист, погода), правая панель (циклическая смена: диагностика -> УВБ-76 -> инфо о фильме). Результат пишется в `fakelogs.txt`.
*   **`clock.py`:** Генерирует массивные ASCII-часы. Использует матричный шрифт, где каждая цифра — это блок 4x3. Поддерживает динамический оффсет для синхронизации времени на экране с задержкой видеотрансляции.
*   **`diag.py`:** Профессиональный модуль мониторинга. 
    *   `run_vmstat`: собирает показатели Steal Time (критично для VPS) и системных прерываний.
    *   `run_mtr`: выполняет трассировку до RTMP-сервера, вычисляя потери пакетов (Loss) и средний пинг на каждом узле.
*   **`uvb.py` :** Генерирует шифровки номерной радиостанции, используя правила лингвистической конкатенации (соединительные гласные).
*   **`res.py`:** Прямой мониторинг `psutil`. Выводит визуальные шкалы (progress bars) нагрузки CPU1/CPU2, занятость RAM и диска, а также статистику облачного хранилища.

### 🎵 Управление контентом и автоматизация
*   **`content.py` (Media Harvester):** Самый сложный модуль сбора. 
    1. Сканирует привязанные Telegram-каналы.
    2. Фильтрует уже скачанные посты через `downloaded.json`.
    3. Выполняет `ffprobe` валидацию: если файл битый или короче 30 секунд, он отбраковывается.
    4. Реализует систему атомарной записи: скачивает в `.part`, проверяет целостность и только потом переименовывает в рабочий файл.
*   **`1h_pl.py` (Playlist Constructor):** Формирует сетку часа. Использует сложный алгоритм ротации: `played` (не играть то, что было в этом цикле) и `recent` (жесткий бан на последние 5 треков). Поддерживает вставку джинглов, рекламы и новостей по шаблону `hour.txt`.
*   **`recorder.py`:** По завершении часа склеивает все файлы через `ffmpeg concat` (без перекодировки, режим `-c copy`) и заливает в Telegram-архив с хэштегами жанра.

### 🎥 VHS-кинотеатр и Видео-пайплайн
*   **`vhs_runner.py`:** Генерирует команды FFmpeg для киносеансов. Накладывает эффект "картинка в картинке" (PiP), где в углу транслируется вспомогательный видеопоток, и бегущую строку внизу экрана.
*   **`vhs_card.py`:** Форматирует метаданные фильма в красивую текстовую карточку: рейтинг IMDb, Rotten Tomatoes, список актеров и описание, обрезанное под ширину интерфейса.
*   **`movie_announcer.py`:** Автоматический промоутер. За 10 минут до начала фильма запрашивает у нейросети (Aria/GPT) генерацию постера в стиле VHS 80-х, скачивает его, конвертирует в JPEG и публикует анонс в Telegram-канал.

## 🛠 3. Технический стек

*   **Язык:** Python 3.10+
*   **Библиотеки:** 
    *   `asyncio` — асинхронное ядро для всех процессов.
    *   `Telethon` — работа с Telegram API (MTProto).
    *   `aiogram` — интерфейс бота-мессенджера.
    *   `psutil` — системные метрики.
    *   `aiohttp` — асинхронные запросы к курсам валют и погоде.
    *   `curses` — для всех встроенных TUI-редакторов.
*   **Внешние зависимости:** 
    *   `FFmpeg` — основной энкодер.
    *   `Streamlink` — для мониторинга живых потоков.
    *   `MTR` / `vmstat` — сетевая и системная диагностика.

## 🖥 4. Инструментарий администратора (TUI Tools)

Проект включает три мощных терминальных приложения для управления без правки JSON/INI вручную:
1.  **`grider.py`:** Визуальный редактор сетки вещания. Позволяет расставлять музыкальные жанры по часам и дням недели простым перемещением курсора.
2.  **`vhs_db_manager.py`:** Полноценная БД фильмов. Поддерживает поиск по API TMDB/OMDB, автоматический перевод описаний через GPT и экспорт базы в CSV.
3.  **`config_editor.py`:** Инструмент для правки `config.ini`. Отображает комментарии к каждому параметру, позволяет менять пути, битрейты и токены в удобном интерфейсе.

## 🚀 5. Развертывание

1.  **Подготовка:** Установите зависимости `apt install ffmpeg mtr-tiny screen`.
2.  **Конфигурация:** Заполните API ключи, создайте телеграм каналы с медиа, выясните их id, получите токены и API ключи и все это суньте в `config.ini`.
3.  **Библиотека:** Создайте папку `pip/` и положите туда несколько MP4 файлов для видео-фона радио.
4.  **Запуск:** Скрипт `autostart.sh` автоматически создаст сессии `screen` для всех компонентов:
    *   `screen -r air` — контроль эфира.
    *   `screen -r status` — мониторинг рендерера.
    *   `screen -r content` — лог загрузчика контента.

## ⚠️ 6. Важные замечания
*   **Цензура:** Модуль `messenger.py` использует `blacklistedwords.txt` для автоматической фильтрации приветов в эфире.
*   **Тайминги:** Переключение эфира настроено на `59:45` каждого часа для синхронизации с реальным временем, так как буфер ffmpeg равен тем же 15 секундам.

## 🛠 7. Внутренний инструментарий (Admin TUI Toolset)

Проект Radio GYCHA укомплектован набором проприетарных терминальных инструментов (TUI), построенных на базе библиотеки `curses`. Они позволяют управлять вещанием в реальном времени, не выходя из консоли сервера.

### 📅 Grider: Архитектор эфирной сетки (`grider.py`)
Этот инструмент предназначен для визуального проектирования недельного расписания. Он исключает ошибки в названиях папок и форматах файлов.

*   **Визуальное поле:** Таблица 7x24, где по горизонтали — дни недели (Пн-Вс), по вертикали — часы (00-23).
*   **Умный выбор:** Вместо ручного ввода названий, Grider предлагает список программ, подтянутый напрямую из `config.ini` (секция `[PROGRAM_DISPLAY_NAMES]`).
*   **VHS Overlay:** Режим предпросмотра (`v`), который подсвечивает блоки, занятые киносеансом, чтобы вы случайно не поставили музыку на время фильма.
*   **Горячие клавиши:**
    *   `Enter`: Выбор жанра из списка.
    *   `d`: Быстрая очистка часа (Silence).
    *   `n`: Создание нового жанра "на лету".
    *   `s`: Синхронизация всех изменений с файлами `program0.txt` ... `program6.txt`.

### 🗃 VHS DB Manager: Интеллектуальная фильмотека (`vhs_db_manager.py`)
Центральный узел управления метаданными для видеосалона. Это не просто редактор JSON, а полноценная станция обработки данных.

*   **Двухпанельный интерфейс:** Слева — список всех кассет в базе (ID поста и название), справа — подробная карточка фильма (год, режиссер, хронометраж, рейтинги).
*   **Интеграция с API:** 
    *   `F3`: Глобальный поиск по базам **TMDB** и **OMDB**. Подтягивает постеры, актерский состав и оригинальные названия.
    *   **IMDb & Rotten Tomatoes:** Автоматическое получение свежих рейтингов для вывода в эфир.
*   **AI-локализация (`F4`):** Интеграция с GPT (через провайдер Aria) для художественного перевода описаний (Overview) с английского на русский.
*   **Экспорт:** Возможность выгрузки всей базы в `vhs_export.csv` для анализа или внешнего редактирования.

### ⚙️ Config Editor: Центр управления системой (`config_editor.py`)
Инструмент для безопасной правки основного конфигурационного файла `config.ini`.

*   **Сохранение структуры:** В отличие от стандартных библиотек Python, этот редактор использует гибридный парсинг, который сохраняет ваши ручные комментарии (строки начинающиеся с `;`) и привязывает их к параметрам.
*   **Контекстные подсказки:** При наведении на параметр (например, `v_bitrate`), в нижней панели отображается описание этого параметра, взятое прямо из комментариев файла.
*   **Многострочный ввод:** Удобный буфер обмена для длинных команд FFmpeg и URL-адресов.
*   **Безопасность:** Проверка типов данных (bool/int/list) перед записью на диск, что предотвращает падение ядра вещания из-за опечатки.

## 🚦 Как запускать
1. Клонируем репозиторий
2. Переходим в папку проекта
3. Выполняем chmod +x setup.sh
4. Выполняем ./setup.sh
5. Дожидаемся разворачивания структуры папок, заполнения мокап-файлами и проверки целостности.
6. Настраиваем точку выхода RTMP
7. Настраиваем облачную часть в конфиге (токены, API, маппинг каналов с содержанием)
8. Если все компоненты на месте, стартуем эфир python3 air_1h.py
9. Если не упало, запускаем остальные компоненты: content.py, watchdog.py (при наличии заполненной локальной бибилиотеки будет работать и без облачной части)



Все утилиты оптимизированы для работы через SSH (рекомендуется использовать `tmux` или `screen` для сохранения сессии):

```bash
# Редактирование сетки вещания
python3 grider.py

# Управление базой фильмов и метаданными
python3 vhs_db_manager.py

# Тонкая настройка системы (битрейты, токены, лимиты)
python3 config_editor.py
```
